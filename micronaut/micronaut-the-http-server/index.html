Variable resolution (变量解析)

Micronaut 尝试通过如下顺序来填充方法参数:

  1. URI 变量, 例如 /{id}
  2. 如果请求是一个 GET 请求, 从请求参数中获取, 例如 ?foo=bar
  3. 如果有一个 @Body 注解并且请求允许这个 body, 那么绑定这个 body 到 @Body 注解修饰的参数上
  4. 如果请求可以有 body 但是没有使用 @Body 注解修饰参数, 那么就尝试解析 body(JSON 或 form data), 将方法参数绑定到 body
  5. 最后, 如果方法参数无法被填充, 那么就返回 400 BAD REQUEST

Custom Argument Binding (自定义参数绑定)

Micronaut uses an ArgumentBinderRegistry to look up ArgumentBinder beans capable of binding to the arguments in controller methods.

Micronaut 使用一个 ArgumentBinderRegistry 来查找能够绑定到 Controller 方法中的参数的 ArgumentBinder Bean

The default implementation looks for an annotation on the argument that is meta-annotated with @Bindable.

默认的实现是先寻找修饰参数的 使用了@Bindable注解修饰的 注解

If one exists the argument binder registry searches for an argument binder that supports that annotation.

如果找到一个使用了 @Bindable 注解修饰的注解, 那么 argument binder registry 将会寻找一个支持这个注解的 argument binder

If no fitting annotation is found Micronaut tries to find an argument binder that supports the argument type.

如果没有找到使用 @Bindable 注解修饰的注解, 那么 Micronaut 将会尝试寻找一个支持参数类型的 argument binder

An argument binder returns a ArgumentBinder.BindingResult.

一个 argument binder 返回一个 ArgumentBinder.BindingResult

The binding result gives Micronaut more information than just the value.

这个 BindingResult 给 Micronaut 提供了更多的信息而不仅仅是值

Binding results are either satisfied or unsatisfied, and either empty or not empty.

BindingResult 不是 satisfied 就是 unsatisfied 的, 不是 empty 就是 not empty 的

If an argument binder returns an unsatisfied result, the binder may be called again at different times in request processing.

如果一个 argument binder 返回一个 unsatisfied 结果, 这个 binder 可能会在处理请求的不同时候被再次调用

Argument binders are initially called before the body is read and before any filters are executed.

argument binders 在 body 被读取和在任何 filter 被执行之前被初始化调用

If a binder relies on any of that data and it is not present, return a ArgumentBinder.BindingResult#UNSATISFIED result.

如果一个 binder 依赖于其中的任何数据, 而该数据不存在, 则返回一个 ArgumentBinder.BindingResult#UNSATISFIED

Returning an ArgumentBinder.BindingResult#EMPTY or satisfied result will be the final result and the binder will not be called again for that request.

返回一个 ArgumentBinder.BindingResult#EMPTY 或者 satisfied result 将是最终的 result, 在这个请求中 binder 不会再次被调用

At the end of processing if the result is still ArgumentBinder.BindingResult#UNSATISFIED, it is considered ArgumentBinder.BindingResult#EMPTY.

在处理的最后如果结果仍然是 ArgumentBinder.BindingResult#UNSATISFIED, 这被认为是 ArgumentBinder.BindingResult#EMPTY

Host Resolution (Host 解析)

Micronaut 提供了一个 HttpHostResolver 接口实现来解析当前 Server 的 host 名称

这个默认实现通过如下地方按顺序来寻找 host 信息:

  1. 提供的配置
  2. Forwarded header
  3. X-Forwarded- headers. 如果 X-Forwarded-Host header 没有提供, 另外的 X-Forwarded headers 被忽略
  4. Host header
  5. 在请求的 URI 上的属性
  6. 在内嵌的 server URI 上的属性
