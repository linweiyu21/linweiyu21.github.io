Configurations

Micronaut 提供了许多内置配置来开启与常见数据库或其他 servers 的集成

Configurations for Reactive Programming (响应式编程的配置)

Project Reactor 被 Micronaut 内置使用

但是要在 Controller 或 HTTP client 方法中使用 Reactor 或其他响应式库 (例如 RxJava) 类型, 需要包含对应的依赖到你的构建中

Reactor Support (Reactor 支持)

要添加 Reactor 的支持 添加如下模块:

implementation("io.micronaut.reactor:micronaut-reactor")

要使用 Reactor HTTP client, 添加如下依赖:

implementation("io.micronaut.reactor:micronaut-reactor-http-client")

查看 Micronaut Reactor 获取更多信息

RxJava 3 Support (RxJava 3 支持)

要添加 RxJava3 的支持 添加如下模块:

implementation("io.micronaut.rxjava3:micronaut-rxjava3")

要使用 RxJava3 HTTP client, 添加如下依赖:

implementation("io.micronaut.rxjava3:micronaut-rxjava3-http-client")

查看 Micronaut RxJava 3 获取更多信息

RxJava 2 Support (RxJava 2 支持)

对 RxJava 1 的遗留支持通过添加如下模块:

implementation("io.micronaut.rxjava1:micronaut-rxjava1")

查看 Micronaut RxJava 1 获取更多信息

Configuring a SQL Data Source (配置一个 SQL 数据源)

可以使用当前默认支持的三种实现之一来配置 JDBC 数据源 - Apache DBCP2, Hikari 和 Tomcat

Configuring a JDBC DataSource (配置一个 JDBC 数据源)

如果你使用 Micronaut CLI 来创建你的项目, 那么你可以添加 jdbc-tomcat,jdbc-hikari,jdbc-dbcp 中的一个特性来预配置一个简单的 JDBC 连接池以及一个默认的 H2 数据库驱动到你的项目中:

To get started, add a dependency for one of the JDBC configurations that corresponds to the implementation you will use. Choose one of the following:

首先, 为其中一个 对应你要使用的实现的 JDBC 配置添加一个依赖, 选择下面中的一个:

runtimeOnly("io.micronaut.sql:micronaut-jdbc-tomcat")

runtimeOnly("io.micronaut.sql:micronaut-jdbc-hikari")

runtimeOnly("io.micronaut.sql:micronaut-jdbc-dbcp")

runtimeOnly("io.micronaut.sql:micronaut-jdbc-ucp")

然后, 添加一个 JDBC 驱动依赖到你的构建中. 例如, 添加 H2 In-Memory Database:

runtimeOnly("com.h2database:h2")

查看 Micronuat SQL libraries 项目的 Configuration JDBC 章节获取更多信息

Configuring Hibernate (配置 Hibernate)

Setting up a Hibernate/JPA EntityManager (配置一个 Hibernate/JPA EntityManager)

如果你使用 Micronaut CLI 创建你的项目, 添加 hibernate-jpa 功能来包含一个 Hibernate JPA 配置到你的项目中:

Micronaut includes support for configuring a Hibernate / JPA EntityManager that builds on the SQL DataSource support.

Micronaut 包括了配置一个构建在 SQL DataSource support 上的 Hibernate/JPA EntityManager 的支持

Micronaut 可以配置一个构建于 SQL DataSource support 的Hibernate/JPA EntityManager

一旦你已经 配置一个或多个数据源 来使用 Hibernate, 那就添加 hibernate-jpa 依赖到你的构建中:

implementation("io.micronaut.sql:micronaut-hibernate-jpa")

Using GORM for Hibernate (为 Hibernate 使用 GORM)

对于 Groovy 用户和熟悉 Grails 框架的用户, 可以使用对 GORM for Hibernate 的特殊支持

要为 Hibernate 使用 GORM 不要包含 Micronaut 内置的 SQL Support 和 hibernate-jpa 依赖, 因为 GORM 本身负责创建 ``DataSource,SessionFactory`等

如果你使用 Micronaut CLI 创建项目, 添加 hibernate-gorm 功能来包含 GORM, 一个基础的连接池配置和一个默认的 H2 数据库驱动到你的项目中:

mn create-app my-app --features hibernate-gorm

Configuring MongoDB (配置 MongoDB)

Setting up the Native MongoDB Driver (配置原生 MongoDB 驱动)

如果你使用 Micronaut CLI 创建项目, 添加 mongo-reactive 功能来配置原生 MongoDB 驱动到你的项目中:

mn create-app my-app --features mongo-reactive

Micronaut 可以自动配置原生 MongoDB Java 驱动. 要使用这个, 添加如下依赖到你的构建中:

implementation("io.micronaut.mongodb:micronaut-mongo-reactive")

然后在 application.yml 中配置 MongoDB server 的 URI:

Example - Configuring a MongoDB server:

mongodb.uri 使用 MongoDB Connection String 格式

然后一个非阻塞的响应式流 MongoClient 就可以在依赖注入中使用

要使用阻塞驱动, 在 mongo-java-driver 上为您的构建添加依赖项:

compile "org.mongodb:mongo-java-client"

然后阻塞的 MongoDB 将会在注入中可用

查看 Micronaut MongoDB 文档获取更多配置与在 Micronaut 中使用 MongoDB 的信息

Configuring Neo4j (配置 Neo4j)

Micronaut 支持为流行的 Neo4j 图形数据库自动配置 Neo4j Bolt 驱动

如果你使用 Micronaut CLI 创建项目, 添加 neo4j-bolt 功能来配置 Neo4j Bolt 驱动到你的项目中:

mn create-app my-app --features neo4j-bolt

要配置 Neo4j Bolt 驱动, 首先添加 neo4j-bolt 模块到你的构建中:

implementation("io.micronaut:micronaut-neo4j-bolt")

然后在 application.yml 中配置 Neo4j server 的 URI:

Example - Configuring neo4j.uri

neo4j.uri 设置的格式必须是 Neo4j 文档的 Connection URIs 章节中描述的格式

一旦上面的配置就绪, 你就可以注入一个 org.neo4j.driver.v1.Driver Bean 实例, 这个 bean 同时提供同步阻塞 API 和基于 CompletableFuture 的非阻塞 API

查看 Micronaut Neo4j 文档获取更多关于配置和在 Micronaut 中使用 Neo4j 的信息

Configuring Postgres (配置 Postgres)

Micronaut 通过 vertx-pg-client 支持流式的非阻塞 client 来连接 Postgres, 这个 client 可以使用一个线程来处理多个数据库连接

Configuring the Reactive Postgres Client (配置流式 Postgres 客户端)

如果你使用 Micronaut CLI 创建项目, 添加 vertx-pg-client 功能来配置响应式 Postgres 客户端到你的项目中:

mn create-app my-app --features vertx-pg-client

要配置流式 Postgres 客户端, 首先添加 vertx-pg-client 模块到你的构建中:

compile "io.micronaut.sql:micronaut-vertx-pg-client"

查看 Micronaut SQL libraries 项目的 Configuring Reactive Postgres 章节获取更多信息

Configuring Redis (配置 Redis)

Micronaut 通过 redis-lettuce 模块为 Redis 自动配置 Lettuce 驱动

Configuring Lettuce

如果你使用 Micronaut CLI 创建项目, 添加 redis-lettuce 功能来配置 Lettuce 驱动到你的项目中:

mn create-app my-app --features redis-lettuce

要配置 Lettuce 驱动, 首先添加 redis-lettuce 模块到你的项目中:

compile "io.micronaut.redis:micronaut-redis-lettuce"

然后在 application.yml 中配置 Redis server 的 URI:

redis.uri 配置必须是 Lettuce wiki 的 Connection URIs 章节中描述的格式

查看 Micronaut Redis 获取更多信息

如果你使用 Micronaut CLI 创建项目, 添加 cassandra 功能来包含 Cassandra 配置到你的项目中:
